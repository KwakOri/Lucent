# Projects Index (프로젝트 목록) - UI Spec

## 경로

`/projects`

## 페이지 목적

- 전체 프로젝트 목록 표시
- 각 프로젝트로 이동할 수 있는 진입점

## 레이아웃 구조

### 1. Page Header

**구성**:
- 페이지 제목: "Projects"
- 서브 타이틀: "Lucent가 기록하는 프로젝트들" (선택)

### 2. Project Cards Grid

**구성**:
- 프로젝트 카드 목록
  - 0th Project (미루루)
  - 1st Project (Drips)

**각 카드 포함 정보**:
- 프로젝트 번호 (0th, 1st, ...)
- 프로젝트 대표 이미지/키비주얼
- 프로젝트 이름
- 한 줄 설명
- 상세 보기 버튼/링크

**레이아웃**:
- 모바일: 1열 (세로 스크롤)
- 태블릿: 2열 그리드
- 데스크톱: 2-3열 그리드

**인터랙션**:
- 카드 전체 클릭 가능
- 호버 시 약간의 확대/그림자 효과 (선택)
- 클릭 → `/projects/{project-id}` 이동

## 상태 관리

### 데이터 로딩
- `projects`: 프로젝트 목록 데이터
- `isLoading`: 로딩 상태
- `error`: 에러 상태

## 데이터 구조

```typescript
interface Project {
  id: string
  name: string
  order: string // "0th", "1st", ...
  description: string
  coverImage: string
  slug: string
}
```

## 디자인 가이드

### 카드 디자인
- 이미지 비율: 16:9 또는 4:3
- 텍스트 정보는 이미지 하단 또는 오버레이
- 일관된 카드 크기 유지

### 호버 효과
- 부드러운 transition
- 과하지 않은 인터랙션

## 반응형

### 모바일 (< 768px)
- 1열 세로 스크롤
- 카드 간격: 16px

### 태블릿 (768px ~ 1024px)
- 2열 그리드
- 카드 간격: 24px

### 데스크톱 (> 1024px)
- 2-3열 그리드
- 카드 간격: 32px
- 최대 너비 제한 (1200px)

## 접근성

- 카드에 적절한 aria-label
- 키보드 네비게이션 지원
- 포커스 스타일 명확히

## 성능

- 이미지 최적화
- 프로젝트 수가 적으므로 페이지네이션 불필요

## 1차 MVP 제외 기능

- 필터/정렬 기능
- 검색 기능
- 아티스트 개별 페이지 진입점
- 프로젝트 타임라인 뷰

## API 연동

- API: `GET /api/projects`
- 응답: 프로젝트 목록 배열
- 정적 데이터로 시작 가능 (추후 API 연동)

## 참고사항

- 현재는 2개 프로젝트만 존재
- 추후 프로젝트 추가 시 확장 가능한 구조
- 단순하고 명확한 정보 전달 우선
