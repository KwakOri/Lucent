# UI – Common Empty State

이 문서는 서비스 전반에서 사용되는 **Empty State UI (데이터 없음 상태)**의 공통 규칙과 UX 정책을 정의한다.

본 문서는 `ui/index.md`, `ui/common/loading.md`, `ui/common/button.md`의 원칙을 따른다.

---

## 1. 역할 정의 (Role)

Empty State는 **표시할 데이터가 없는 상태**를 사용자에게 명확하고 긍정적으로 전달하기 위한 UI이다.

### Empty State의 목적

- 사용자의 혼란 감소
- 현재 상태에 대한 명확한 설명 제공
- 다음 행동을 자연스럽게 유도

---

## 2. Empty State가 필요한 경우

- 리스트 데이터가 0건인 경우
- 검색 결과가 없는 경우
- 최초 진입 상태 (아직 데이터 없음)

예시:

- 장바구니가 비어 있음
- 주문 내역이 없음
- 관심 상품이 없음

---

## 3. Empty vs Error 구분

| 구분      | Empty State | Error State |
| --------- | ----------- | ----------- |
| 의미      | 정상 상태   | 비정상 상태 |
| UX 톤     | 안내 / 유도 | 사과 / 설명 |
| 다음 행동 | 선택 사항   | 조치 필요   |

> Empty는 오류가 아니다.

---

## 4. 구성 요소

Empty State는 다음 요소로 구성될 수 있다:

- 아이콘 또는 일러스트 (선택)
- 제목 메시지 (필수)
- 보조 설명 텍스트 (선택)
- CTA 버튼 (선택)

---

## 5. 메시지 톤 가이드

- 친절하고 긍정적인 표현 사용
- 사용자를 탓하지 않는다
- 현재 상황을 담담하게 설명한다

예시:

- ⭕ 아직 담긴 상품이 없어요
- ⭕ 주문 내역이 아직 없어요
- ❌ 데이터가 존재하지 않습니다

---

## 6. CTA(Button) 사용 기준

CTA는 **다음 행동이 명확할 때만 제공**한다.

허용 예:

- "상품 보러 가기"
- "검색 다시 하기"

지양 예:

- 의미 없는 닫기 버튼
- 과도한 CTA

---

## 7. Mobile UX 고려

- 화면 중앙 정렬을 기본으로 한다.
- 스크롤 없이 인지 가능해야 한다.
- CTA는 한 개를 권장한다.

---

## 8. 접근성 (Accessibility)

- 메시지는 스크린 리더에 명확히 전달되어야 한다.
- 아이콘/일러스트는 보조적 요소로 처리한다.

---

## 9. Loading과의 관계

- 데이터 로딩 중에는 Empty State를 표시하지 않는다.
- Loading 완료 후 데이터가 없을 경우에만 표시한다.

---

## 10. 확장 고려

- 상황별 Empty State 변형
- 일러스트 테마 통일
- 사용자 행동에 따른 동적 메시지

---

### 요약

- Empty State는 정상 UX 흐름의 일부
- 긍정적이고 친절한 톤 유지
- 다음 행동을 자연스럽게 안내
- Mobile First 중앙 배치
