# Home (메인 페이지) - UI Spec

## 경로

`/`

## 페이지 목적

- Lucent Management의 정체성 전달
- 현재 진행 중인 굿즈 홍보
- 소속 프로젝트 및 아티스트 소개
- 레이블 소개 및 공식 SNS 연결

## 레이아웃 구조

### 1. Hero Section

**목적**: 레이블 슬로건 전달 + 굿즈 홍보

**구성**:
- 슬로건 텍스트: "숨겨진 감정과 목소리가 자연스럽게 드러나는 순간을 기록하는 레이블"
- 배경 이미지/비디오 (선택)
- 자동 슬라이드 기능
  - 첫 화면: 슬로건
  - 이후: 굿즈 홍보 슬라이드 (3-5초 간격)

**인터랙션**:
- 자동 슬라이드 (3-5초)
- 수동 좌우 네비게이션 (선택)
- 슬라이드 인디케이터 (점)

### 2. Projects Preview Section

**목적**: 소속 프로젝트 및 아티스트 간단 소개

**구성**:
- 섹션 제목: "소속 아티스트" 또는 "Projects"
- 프로젝트 카드 목록 (2개)
  - 0th Project: 미루루
  - 1st Project: Drips (2명)
- 각 카드 포함 정보:
  - 프로젝트 이름
  - 대표 이미지
  - 한 줄 설명
  - 상세 보기 버튼

**레이아웃**:
- 모바일: 세로 스크롤
- 데스크톱: 가로 2열 그리드

**인터랙션**:
- 카드 클릭 → `/projects/{project}` 이동

### 3. About Lucent Section

**목적**: 레이블 정체성 설명

**구성**:
- 섹션 제목: "Lucent Management"
- 설명 텍스트:
  - "우리는 버츄얼 MCN이 아니라 매니지먼트 레이블입니다"
  - 프로젝트를 기록하고 관리하는 레이블
  - 감성적이고 간결한 톤

**레이아웃**:
- 중앙 정렬 텍스트
- 배경색 또는 구분선으로 섹션 구분

### 4. Social Link Section

**목적**: 공식 트위터 연결

**구성**:
- 섹션 제목: "Follow Us"
- 트위터 아이콘 + 링크
- (선택) 최근 트윗 임베드

**인터랙션**:
- 클릭 시 새 창으로 트위터 오픈

## 상태 관리

### 슬라이드 상태
- `currentSlide`: 현재 슬라이드 인덱스
- `autoPlay`: 자동 재생 여부
- `slides`: 슬라이드 데이터 배열

### 프로젝트 데이터
- API 또는 정적 데이터에서 로드
- 로딩/에러 상태 처리

## 디자인 가이드

### 톤앤매너
- 감성적, 섬세함
- 기록의 가치를 느낄 수 있는 디자인
- 과하지 않은 애니메이션

### 색상
- 메인 컬러: 레이블 아이덴티티 컬러
- 배경: 밝은 톤 또는 다크 모드 지원

### 타이포그래피
- 슬로건: 큰 폰트, 감성적 폰트
- 본문: 가독성 좋은 산세리프

## 반응형

### 모바일 (< 768px)
- 전체 섹션 세로 스크롤
- Hero 슬라이드 전체 화면
- 프로젝트 카드 1열

### 태블릿 (768px ~ 1024px)
- Hero 슬라이드 유지
- 프로젝트 카드 2열

### 데스크톱 (> 1024px)
- Hero 슬라이드 높이 조정
- 프로젝트 카드 2열 또는 3열

## 접근성

- 슬라이드 자동 재생 일시정지 버튼
- 키보드 네비게이션 지원
- 이미지 alt 텍스트
- 적절한 heading 레벨

## 성능

- 이미지 최적화 (Next.js Image)
- 슬라이드 lazy loading
- 초기 로딩 속도 우선

## 참고사항

- 1차 MVP에서는 과도한 애니메이션 제외
- 프로젝트 연표는 제외
- 단순하고 명확한 정보 전달에 집중
