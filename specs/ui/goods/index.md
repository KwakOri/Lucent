# Goods Hub (굿즈 허브) - UI Spec

## 경로

`/goods`

## 페이지 목적

- 아티스트별 굿즈샵 진입점 제공
- 각 아티스트의 굿즈샵으로 연결

## 레이아웃 구조

### 1. Page Header

**구성**:
- 페이지 제목: "Goods Shop"
- 서브 타이틀: "아티스트별 굿즈를 만나보세요" (선택)

### 2. Artist Shop Cards

**구성**:
- 아티스트별 굿즈샵 카드
  - 미루루
  - Drips (2차 확장)

**각 카드 포함 정보**:
- 아티스트 이름
- 대표 이미지
- 샵 컨셉 미리보기 (색상/분위기)
- "샵 보러가기" 버튼

**레이아웃**:
- 모바일: 1열 세로 스크롤
- 데스크톱: 2열 그리드

**인터랙션**:
- 카드 클릭 → `/goods/{artist-slug}` 이동

## 상태 관리

### 데이터 로딩
- `artists`: 아티스트 목록 (굿즈샵 정보 포함)
- `isLoading`: 로딩 상태
- `error`: 에러 상태

## 데이터 구조

```typescript
interface ArtistShop {
  id: string
  name: string
  slug: string
  description: string
  coverImage: string
  themeColor: string
  isActive: boolean
}
```

## 디자인 가이드

### 카드 디자인
- 각 아티스트의 샵 테마를 미리 느낄 수 있는 디자인
- 미루루: 파스텔 하늘색 배경/테두리
- Drips: (추후 지정)

### 호버 효과
- 부드러운 transition
- 카드 확대 또는 그림자 효과

## 반응형

### 모바일 (< 768px)
- 1열 세로 스크롤
- 카드 간격: 16px
- 카드 전체 너비

### 태블릿 (768px ~ 1024px)
- 2열 그리드
- 카드 간격: 24px

### 데스크톱 (> 1024px)
- 2-3열 그리드
- 카드 간격: 32px
- 최대 너비 제한 (1200px)

## 접근성

- 카드에 적절한 aria-label
- 키보드 네비게이션 지원
- 포커스 스타일 명확히

## 성능

- 이미지 최적화
- 아티스트 수가 적으므로 간단한 구조 유지

## 1차 MVP 제외 기능

- 통합 상품 리스트
- 검색 기능
- 필터/정렬 기능
- 카테고리 분류

## API 연동

- API: `GET /api/artists?hasShop=true`
- 응답: 굿즈샵이 있는 아티스트 목록
- 정적 데이터로 시작 가능

## 참고사항

- 1차 MVP에서는 미루루 굿즈샵만 활성화
- Drips 굿즈샵은 "준비 중" 또는 비활성 상태로 표시
- 추후 아티스트 추가 시 확장 가능한 구조
